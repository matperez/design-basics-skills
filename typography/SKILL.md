# Typography — процесс подбора шрифтов

Этот скилл — пошаговый процесс, в результате которого агент выбирает шрифтовой набор (1–3 шрифта) под конкретную задачу клиента. Не справочник «что такое кернинг», а дерево решений.

## Шаг 1. Собрать вводные

Прежде чем открывать каталог шрифтов, уточни у клиента (или выведи из контекста):

| Вопрос | Зачем |
|--------|-------|
| **Носитель** — экран (сайт, приложение, презентация) или печать (флаер, визитка, книга)? | Определяет RGB vs CMYK, web-safe, лицензии |
| **Тип контента** — длинный текст, короткие заголовки, UI-элементы, логотип? | Определяет класс шрифта и количество начертаний |
| **Тон бренда** — строгий, тёплый, игривый, премиальный, tech, ретро? | Сужает выбор до 2–3 подкатегорий |
| **Аудитория** — возраст, культура, уровень зрения, устройства | Влияет на размеры, контраст, accessibility |
| **Ограничения** — бюджет (бесплатные / коммерческие), существующий brand book, дизайн-система | Фильтрует каталог |

Если клиент не знает тон бренда, предложи описать его 3–5 прилагательными и подбери из таблицы в шаге 2.

## Шаг 2. Определить класс шрифта

По тону бренда и типу контента выбери **основной класс** (для body / основного текста) и **акцентный класс** (для заголовков / логотипа).

### Матрица «тон → класс»

| Тон бренда | Body (основной) | Headline / акцент |
|------------|-----------------|-------------------|
| **Строгий, корпоративный** | Sans serif (нейтральный) | Serif или тот же sans bold |
| **Тёплый, дружелюбный** | Sans serif (humanist) | Serif (rounded) или sans bold |
| **Премиальный, luxury** | Serif (transitional/modern) | Display serif или serif bold |
| **Tech, стартап** | Sans serif (geometric) | Mono или display sans |
| **Творческий, арт** | Sans serif или serif | Display, script (только акцент) |
| **Ретро, винтаж** | Slab serif или serif | Display, script |
| **Игривый, детский** | Sans serif (rounded) | Display (rounded), script |
| **Минималистичный** | Sans serif (geometric, чистый) | Тот же sans в другом весе |

### Классы шрифтов — краткая справка

| Класс | Когда | Ограничения |
|-------|-------|-------------|
| **Serif** | Авторитет, традиция, длинные тексты, editorial | На мелких экранах засечки могут «забиваться» |
| **Sans serif** | Универсал: UI, экран, body, заголовки | Менее «характерный» — нужна пара для контраста |
| **Script** | Только короткие акценты: подпись, логотип, цитата | Нечитабелен в body; max 3–5 слов |
| **Monospace** | Код, данные, ретро, tech-акцент | Занимает больше места; не для длинного body |
| **Display** | Логотипы, постеры, крупные заголовки | Не для body text; часто 1–2 начертания |
| **Slab serif** | Ретро, заголовки, бренды с «тяжёлым» характером | Может доминировать; баланс с лёгким body |

## Шаг 3. Подобрать конкретные шрифты

Открой `references/font-catalog.md` — там шрифты сгруппированы по use case с характеристиками и рекомендациями «Best for». Используй как lookup-таблицу.

**Правило выбора:** предложи 2–3 варианта на каждую роль (body, headline), кратко обоснуй выбор клиенту.

### Быстрые рекомендации по ролям

**Body text для экрана (длинный текст):**
Inter, Roboto, Open Sans, Source Sans Pro, Lato, Work Sans, DM Sans, Noto Sans.

**Body text для печати:**
Merriweather, Source Serif 4, EB Garamond, Libre Baskerville, Roboto Serif.

**Заголовки (нейтральные):**
Poppins Bold, Raleway SemiBold, Montserrat Bold, Plus Jakarta Sans Bold.

**Заголовки (характерные):**
Playfair Display, Abril Fatface, Instrument Serif, Bodoni Moda, Oswald.

**Логотип:**
Выбор зависит от типа логотипа (wordmark, letterform, monogram). См. `references/font-catalog.md`, секция «Логотипы».

**UI-элементы (кнопки, метки, навигация):**
Inter, Roboto, DM Sans, Public Sans, Work Sans — нейтральные, читабельные в мелком кегле.

## Шаг 4. Составить пару

Пара = **контраст ролей, а не конфликт стилей**. Одни шрифт — «рабочий», другой — «акцентный».

### Правила пары

1. **Контраст структуры** — serif + sans, display + neutral, mono + humanist.
2. **Максимум 2–3 шрифта.** Body + headline. Третий — только если есть отдельная роль (код, цитата, подпись).
3. **Согласованная x-height** — если высота строчных букв сильно различается, пара будет выглядеть рассогласованно.
4. **Баланс веса** — жирный headline требует лёгкого body. Два тяжёлых = конкуренция за внимание.
5. **Проверка на реальном контенте** — не на lorem ipsum. Подставить настоящие заголовки и абзацы.

### Проверенные пары (быстрый справочник)

| Headline | Body | Характер | Сценарий |
|----------|------|----------|----------|
| Playfair Display | Source Sans Pro | авторитетный + нейтральный | блоги, журналы, editorial |
| Abril Fatface | Lato | элегантный + чистый | luxury, fashion |
| Oswald | Noto Sans | плакатный + универсальный | events, posters |
| Poppins Bold | Inter | округлый + функциональный | SaaS, dashboards |
| Raleway | Merriweather | современный + классический | education, corporate |
| Space Mono | Plus Jakarta Sans | retro-tech + дружелюбный | tech, portfolio |
| Instrument Serif | DM Sans | современная классика + clean | branding, agency |
| Fugaz One | Work Sans | энергичный + профессиональный | marketing, tech |

Полный каталог пар с обоснованием: `references/font-catalog.md`.

## Шаг 5. Настроить типографическую систему

После выбора шрифтов — задать параметры. Ниже стартовые значения; каждый шрифт требует подгонки.

### Размеры (type scale для экрана)

| Роль | Размер | Вес | Примечание |
|------|--------|-----|-----------|
| H1 | 36–48px | Bold (700) | Одна мысль на страницу |
| H2 | 24–32px | SemiBold (600) | Секции |
| H3 | 20–24px | Medium (500) | Подразделы |
| Body | 16–18px | Regular (400) | Базовый текст |
| Small / caption | 12–14px | Regular (400) | Метки, подписи |
| Button / CTA | 14–16px | Medium–Bold (500–700) | Чуть крупнее body для тап-зоны |

### Spacing

| Параметр | Стартовое значение | Когда менять |
|----------|-------------------|-------------|
| **Line-height (leading)** | 1.4–1.6 × font-size для body | Уменьшить для крупных заголовков (1.1–1.2) |
| **Letter-spacing (tracking)** | 0 для body | +0.02–0.05em для ALL CAPS; может потребоваться −0.01 для крупных заголовков |
| **Paragraph spacing** | 0.5–1.0 × font-size | Зависит от плотности контента |

### Kerning

- **Логотипы и заголовки** — всегда проверять вручную (manual kerning).
- **Body text** — metric или optical kerning из шрифта; ручной не нужен.
- **Проблемные пары** — AV, AW, To, Ty, Wa, We, Yo — проверять визуально.

## Шаг 6. Проверить и валидировать

Перед финальной рекомендацией пройди чеклист:

- [ ] **Читабельность:** body text 16px+ на экране, 10pt+ в печати. Попробуй прочитать абзац — легко?
- [ ] **Контраст:** текст и фон проходят WCAG AA (4.5:1 для обычного текста, 3:1 для крупного).
- [ ] **Мобильный тест:** заголовки не обрезаются, body читается без zoom.
- [ ] **Иерархия:** закрой глаза, открой — видно ли, что важнее? H1 > H2 > body очевидно?
- [ ] **Консистентность:** одинаковые элементы = одинаковые стили. Нет «случайных» размеров.
- [ ] **Лицензия:** подходит для целевого использования (web, app, print, коммерческое).
- [ ] **Веб-оптимизация:** если для сайта — шрифт доступен через Google Fonts или CDN; variable font предпочтительнее.
