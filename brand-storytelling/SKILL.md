# Brand & Storytelling — процесс построения и коммуникации бренда

Этот скилл — пошаговый процесс: от определения идентичности бренда до её визуального воплощения (логотип, style guide) и донесения до аудитории (сторителлинг, презентации). Не справочник «что такое emblem», а дерево решений.

## Шаг 1. Определить идентичность бренда

Прежде чем рисовать логотип или выбирать цвета, зафиксируй фундамент. Уточни у клиента (или выведи из контекста):

| Вопрос | Зачем | Пример |
|--------|-------|--------|
| **Миссия** — какую проблему решаете? | Ядро нарратива | «Помогаем малому бизнесу продавать онлайн без кода» |
| **Ценности** — 3–5 слов, которые определяют бренд | Фильтр для всех решений | Простота, честность, скорость |
| **Личность бренда** — 3–5 прилагательных | Задаёт тон визуала и копирайтинга | «Современный, дружелюбный, надёжный» |
| **Целевая аудитория** — кто они, что им важно? | Определяет стиль и язык | Владельцы малого бизнеса 25–45, ценят простоту |
| **Конкуренты** — как выглядят, как звучат? | Найти пространство для отстройки | «Конкуренты все синие и корпоративные — есть шанс быть тёплым» |
| **Носители** — где бренд живёт? | Digital-only vs omnichannel vs print | Сайт + приложение + соцсети |

Если клиент не может описать личность — предложи выбрать из пар:
- Строгий ↔ Игривый
- Классический ↔ Современный
- Доступный ↔ Премиальный
- Минималистичный ↔ Насыщенный
- Корпоративный ↔ Человечный

Результат шага: **бренд-бриф** — 1 абзац, который может прочитать любой член команды и понять, «кто мы и для кого».

## Шаг 2. Спроектировать визуальный знак (логотип)

### 2.1. Выбрать тип логотипа

По характеру бренда и контексту использования выбери тип из матрицы:

| Тип | Что это | Когда выбирать | Примеры |
|-----|---------|----------------|---------|
| **Wordmark** | Название бренда = логотип (типографика) | Короткое запоминающееся имя; хочется максимум узнаваемости имени | Netflix, Disney, Visa, Google |
| **Letterform** | Одна стилизованная буква | Бренд уже известен; нужна компактная иконка | McDonald's (M), Facebook (f) |
| **Monogram** | Инициалы компании как знак | Luxury, наследие, длинное название | LV, CNN, Chanel, IBM |
| **Abstract** | Геометрические формы без букв | Современный, tech, концептуальный образ | Pepsi, Adidas, Toyota |
| **Pictorial mark** | Узнаваемый символ / иконка | Бренд достаточно известен, чтобы символ работал без текста | Apple, Target, Twitter |
| **Combination** | Знак + текст | Гибкость: вместе и раздельно; для новых брендов | Lacoste, Burger King, Airbnb |
| **Emblem** | Текст внутри формы (щит, круг) | Наследие, традиция, «гербовый» характер | Starbucks, BMW, Harley-Davidson |
| **Mascot** | Персонаж как лицо бренда | Детская аудитория, food, спорт, эмоциональная связь | KFC, Duolingo, Pringles |
| **Dynamic** | Меняется в зависимости от контекста | Digital-first, медиа, адаптивный бренд | Google Doodles, Spotify, MTV |

**Если сомневаешься:** для нового бренда **combination mark** — самый безопасный старт (символ работает и с текстом, и без).

### 2.2. Подобрать цвет и шрифт для логотипа

- **Цвет:** применяй процесс из скилла `color-theory` — шаги 1–4.
- **Шрифт:** применяй процесс из скилла `typography` — шаги 1–4.
- **Для wordmark / letterform / monogram** шрифт = самый важный элемент логотипа.
- **Формы:** круг → единство, доверие; квадрат → стабильность; треугольник → динамика, инновация; органические кривые → движение, мягкость.

### 2.3. Итерации и тестирование

1. Создай 3–5 вариантов.
2. Проверь на масштабируемость: favicon (16px) → соцсети (avatars) → сайт → билборд.
3. Проверь в ч/б — логотип должен работать без цвета.
4. Собери фидбек у целевой аудитории (A/B тест, опрос, peer review).
5. Финализируй: подготовь версии для digital (SVG, PNG) и print (AI, EPS, PDF).

### 2.4. Принципы хорошего логотипа (чеклист)

- [ ] **Простота** — узнаваем за доли секунды.
- [ ] **Масштабируемость** — от favicon до баннера.
- [ ] **Уникальность** — не путается с конкурентами.
- [ ] **Вневременность** — не привязан к текущему тренду.
- [ ] **Уместность** — соответствует отрасли и аудитории.
- [ ] **Читаемость** — текст читаем в любом размере.

Подробнее о 12 типах с иллюстрациями: `references/logo-types-guide.md`.

## Шаг 3. Построить визуальную систему (style guide)

Style guide — источник правды для всех, кто работает с брендом. Он фиксирует «как мы выглядим и звучим».

### 5 компонентов style guide

| Компонент | Что включает | Зачем |
|-----------|-------------|-------|
| **Brand story** | Миссия, видение, ценности, обещание | Контекст для всех решений |
| **Целевая аудитория** | Персоны, демография, потребности | Все решения — для них |
| **Визуальная идентичность** | Логотип (правила использования, запретные варианты), цвета (HEX, RGB, CMYK), типографика (семейства, размеры, иерархия), иконографика, стиль фото | Консистентность дизайна |
| **Голос бренда** | Прилагательные тона (modern, clever, caring), различия по каналам (Instagram vs LinkedIn) | Единый тон коммуникации |
| **Гайдлайны по тексту** | Уровень читаемости, грамматика, бренд-слова, запретные формулировки | Консистентность копирайтинга |

### Процесс создания

1. **Outline** — оглавление + структура (25–35 страниц — ориентир).
2. **Сбор гайдлайнов** — визуал + копирайтинг + accessibility.
3. **Ревью** — стейкхолдеры внутри и вне бренд-команды.
4. **Публикация** — доступен всем (дизайнеры, маркетинг, HR, фрилансеры).
5. **Обновление** — каждые 6–12 месяцев; style guide — живой документ.

### Правила для style guide

- Включай **dos and don'ts** с реальными примерами.
- Делай интерактивным — возможность задать вопрос или оставить запрос.
- Лого: покажи допустимые зоны (clear space), минимальные размеры, запретные модификации.

## Шаг 4. Применить сторителлинг к продукту

Customer journey = сюжет. Пользователь — главный герой. Задача дизайнера — выстроить историю так, чтобы герой достиг цели.

### Customer journey как нарратив

| Глава | Вопрос пользователя | Что проектируем |
|-------|---------------------|-----------------|
| **Вход** | «Что это? Это мне?» | Ценностное предложение, первый экран |
| **Знакомство** | «Куда идти? Что тут есть?» | Навигация, витрина, категории |
| **Действие** | «Как это сделать?» | Путь: выбор → настройка → корзина → оформление |
| **Доверие** | «Могу ли доверять?» | Отзывы, гарантии, безопасность (в нужный момент!) |
| **Завершение** | «Всё прошло хорошо?» | Подтверждение, отслеживание, поддержка |
| **Возвращение** | «Стоит ли вернуться?» | Email, рекомендации, программа лояльности |

### Принципы продуктового сторителлинга

- **Прогрессивное раскрытие** — не вываливай всё на одном экране. Детали про доставку, безопасность, возврат — когда пользователь к ним подойдёт.
- **Гостеприимство** — представь продукт и обещание ценности одной ясной фразой.
- **Каждый элемент — часть истории** — если элемент не служит нарративу, он мешает. Убрав лишнее, усиливаешь важное.
- **Нарратив для команды** — сторителлинг помогает защитить дизайн-решения перед стейкхолдерами: «Мы сделали так, потому что на этом этапе пользователь спрашивает себя X».

Подробнее: `references/storytelling-in-design.md`.

## Шаг 5. Донести бренд через презентацию

Когда нужно представить бренд, продукт или идею аудитории.

### Структура презентации (классическая 5-актная)

1. **Intro** — зацепи внимание (hook), представься.
2. **Восходящее действие** — контекст, проблема, данные.
3. **Кульминация** — главная идея / решение / продукт.
4. **Возврат** — как это реализовать, next steps.
5. **Заключение** — summary, CTA.

### 6 типов hooks для открытия

| Тип hook | Пример | Когда работает |
|----------|--------|---------------|
| **Разрушение мифа** | «Всё, что вы знаете о X — неправда» | Нужно перевернуть устоявшееся мнение |
| **Провокационная статистика** | «87% пользователей уходят в первые 10 секунд» | Есть яркая цифра, подтверждающая проблему |
| **Вовлекающий вопрос** | «Поднимите руку, кто…» | Маленькая аудитория, нужно вовлечь сразу |
| **Личная история** | «Когда я впервые…» | Нужна эмпатия и доверие |
| **Визуальный удар** | Чёрный экран → одно мощное изображение | Тема лучше показывается, чем рассказывается |
| **Цитата** | «Design is how it works — Steve Jobs» | Авторитет усиливает тезис |

### Дизайн слайдов — быстрые правила

- **Правило 7×7:** максимум 7 строк, 7 слов на строку. Лучше — меньше.
- **Один слайд = одна идея.** Сложные слайды = потеря внимания.
- **60-30-10 для цвета** — dominant, secondary, accent (применяй скилл `color-theory`).
- **Типографика** — единая пара шрифтов (headline + body), размеры consistency (применяй скилл `typography`).
- **White space** — не бойся пустоты; она направляет внимание.
- **Визуалы > текст** — диаграммы, скриншоты, прототипы побеждают списки.

### Доставка — советы для уверенного выступления

- Выучи первые 60 секунд наизусть — нервы максимальны в начале.
- Пауза 3 секунды после hook — дай зацепке «осесть».
- Взгляд: не скани толпу — смотри на одного человека до конца мысли, потом переключись.
- Не стой столбом, но и не мельтеши — двигайся с целью.

Подробнее: `references/presentation-playbook.md`.

## Шаг 6. Резюме как бренд-артефакт (если клиент — персона)

Резюме — это личный бренд на одной странице. Те же принципы:

1. **Контент первичен** — сначала напиши всё, потом оформляй.
2. **Иерархия** — самое релевантное вверху. Gestalt: proximity группирует опыт, similarity объединяет роли.
3. **Типографика** — передаёт личность. Не перегружай: 1–2 шрифта.
4. **Одна страница** — если нет 20+ лет опыта.
5. **Адаптируй под вакансию** — таргетируй, как любой бренд таргетирует аудиторию.
