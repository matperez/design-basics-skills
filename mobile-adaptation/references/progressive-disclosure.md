# Progressive Disclosure: когда и как скрывать контент на мобильных

## Определение

Progressive disclosure — стратегия управления сложностью информации: показываем только то, что нужно **в этой точке взаимодействия**, а расширенное раскрываем по запросу пользователя.

> «Show less, provide more.» — Nick Babich, UX Planet

## Когда применять

| Ситуация | Паттерн раскрытия |
|----------|-------------------|
| **FAQ с 5+ вопросами** | Аккордеон; первые 2–3 раскрыты, остальные свёрнуты |
| **Подробные характеристики / спецификации** | Раскрываемый блок «Подробнее» или «Все характеристики» |
| **Длинный список услуг / карточек** | Показать 3–4 + кнопка «Все N услуг» |
| **Юридические условия, disclaimer** | Свёрнут по умолчанию; заголовок-ссылка достаточно видим |
| **Детальные биографии, кейсы** | Краткое резюме (2–3 строки) + «Читать полностью» |
| **Вторичная навигация** | Гамбургер-меню, а не 8 пунктов в строку |
| **Расширенные настройки, фильтры** | Базовые видны; «Ещё фильтры» для остального |

## Когда НЕ применять

| Ситуация | Почему не прятать |
|----------|-------------------|
| **Основной CTA** | Потеря конверсии; должен быть видим всегда |
| **Цена / стоимость** | Критически важно для решения; скрытие создаёт недоверие |
| **Контактная информация** | Частый запрос мобильных пользователей (микромомент «I-want-to-go») |
| **Первичное ценностное предложение** | Ради этого открыли страницу |
| **Единственный путь к важной информации** | Доступность + доверие + юридические риски |

## Главная опасность

> Неверно угадать, что «вторично». Тогда пользователь не найдёт критичное.

Решение: опираться на **данные** (аналитика, поиск по сайту, интервью), а не на предположение дизайнера.

Как предупредил Jakob Nielsen:
> «One of usability's most hard-earned lessons is that you are not the user.»

## Паттерны реализации

### Аккордеон
- Заголовок-вопрос виден всегда; содержимое разворачивается по тапу.
- Показать шеврон или «+» как подсказку об интерактивности.
- Не прятать в аккордеон то, что нужно **каждому** посетителю.

### «Подробнее» / «Читать дальше»
- Показать первые 2–3 строки + truncation.
- Кнопка «Подробнее» раскрывает inline; не уводить на другую страницу без необходимости.

### Tabs / Chips
- 3–5 вкладок для категорий; содержимое меняется без перезагрузки.
- На мобиле горизонтальный скролл табов допустим при наличии visual cue.

### Bottom Sheet / Drawer
- Детальная информация выезжает снизу; контекст основного экрана сохраняется.
- Подходит для фильтров, карт, развёрнутых описаний.

## Техническая заметка: SEO и доступность

- Свёрнутый контент должен **оставаться в DOM** (не `display:none` без aria или JS-toggle).
- Google Mobile-first indexing: контент в аккордеонах, табах и `<details>` **индексируется** как обычный текст.
- Для screen readers: использовать `<details>`/`<summary>`, `aria-expanded`, `aria-controls`.
