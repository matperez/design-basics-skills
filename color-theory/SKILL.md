# Color theory — процесс построения палитры

Этот скилл — пошаговый процесс, в результате которого агент строит цветовую палитру (3–6 цветов + нейтральные) под конкретную задачу клиента. Не справочник «что такое комплементарные цвета», а дерево решений.

## Шаг 1. Собрать вводные

Прежде чем открывать цветовой круг, уточни у клиента (или выведи из контекста):

| Вопрос | Зачем |
|--------|-------|
| **Носитель** — экран (сайт, приложение) или печать (флаер, упаковка)? | RGB vs CMYK; gamut отличается |
| **Тип продукта** — что именно (SaaS, e-commerce, портфолио, мобильное приложение, логотип)? | Определяет количество цветов и роли |
| **Настроение бренда** — строгий, тёплый, игривый, премиальный, природный, tech? | Сужает температуру и насыщенность |
| **Аудитория** — возраст, культурный контекст, уровень зрения | Влияет на accessibility и символику |
| **Есть ли существующий бренд-цвет?** | Если да — он становится базовым, остальное строится от него |
| **Ограничения** — дизайн-система, тёмная/светлая тема, совместимость с партнёрскими брендами | Фильтрует выбор |

Если клиент не знает настроение — предложи описать его 3–5 прилагательными и подбери по матрице в шаге 2.

## Шаг 2. Выбрать базовый цвет

Базовый цвет — якорь всей палитры, он задаёт тон бренда.

### Матрица «настроение → цвет»

| Настроение | Рекомендуемые цвета | Примеры брендов |
|------------|-------------------|-----------------|
| **Доверие, надёжность, профессионализм** | Синий, тёмно-синий | IBM, Zoom, PayPal |
| **Энергия, страсть, срочность** | Красный, оранжевый | Coca-Cola, Red Bull, DoorDash |
| **Спокойствие, природа, здоровье** | Зелёный, мятный, sage | Starbucks, Spotify, Whole Foods |
| **Luxury, премиальность** | Чёрный, тёмно-фиолетовый, золотой | Chanel, Mastercard Black |
| **Оптимизм, дружелюбие, молодость** | Жёлтый, оранжевый | Snapchat, McDonald's |
| **Творчество, уникальность** | Фиолетовый, пурпурный | Twitch, Cadbury |
| **Чистота, минимализм** | Белый, светло-серый | Apple, Google |
| **Тепло, уют, природность** | Коричневый, терракота, бежевый | UPS, Timberland |
| **Романтика, игривость** | Розовый, коралловый | Barbie, Dribbble |

### Психология цвета — краткая справка

| Цвет | Позитивные ассоциации | Негативные ассоциации | Культурные нюансы |
|------|----------------------|----------------------|-------------------|
| Красный | Любовь, энергия, действие | Опасность, агрессия | В Китае — удача и процветание |
| Синий | Доверие, спокойствие, безопасность | Холодность, дистанция | Универсально позитивный |
| Зелёный | Рост, природа, здоровье | Зависть, незрелость | Священный в исламе |
| Жёлтый | Радость, оптимизм, внимание | Тревога, дешевизна | Низкий контраст — проблемы с accessibility |
| Фиолетовый | Роскошь, мудрость, креатив | Высокомерие, помпезность | Исторически — цвет власти |
| Оранжевый | Тепло, дружелюбие, энтузиазм | Дешевизна при перенасыщении | — |
| Розовый | Романтика, мягкость, игривость | Незрелость | Культурно изменчив (гендер) |
| Чёрный | Элегантность, сила, таинственность | Тяжесть, мрачность | Траур (Запад) |
| Белый | Чистота, простота, свежесть | Стерильность, пустота | Траур (Восток) |
| Коричневый | Стабильность, природность, уют | Скука, старомодность | — |

**Важно:** символика цвета контекстуальна по культуре — не абсолютна. Всегда проверяй для целевой аудитории.

## Шаг 3. Выбрать схему гармонии

По типу проекта и нужному уровню контраста выбери схему. Базовый цвет уже определён — теперь строим палитру вокруг него.

### Матрица «задача → схема»

| Задача / контекст | Рекомендуемая схема | Почему |
|-------------------|-------------------|-------|
| Минималистичный UI, banking, productivity | **Monochromatic** | Чисто, невозможно столкнуть цвета |
| Wellness, spa, organic, натуральные продукты | **Analogous** | Мягко, природно, не раздражает |
| CTA-ориентированный лендинг, спорт, внимание | **Complementary** | Максимальный контраст, но нужна дисциплина |
| Детские приложения, creative tools | **Split-complementary** | Контраст мягче, больше разнообразие |
| Игровые платформы, яркие бренды | **Triadic** | Энергичная, требует баланса: один доминирует |
| Сложные UI с множеством состояний | **Tetradic / Square** | Максимальное разнообразие; нужны нейтральные |

### 7 схем — краткая механика

| # | Схема | Как строить | Количество цветов |
|---|-------|-------------|-------------------|
| 1 | **Monochromatic** | Один hue → tint, shade, tone | 3–5 (один hue) |
| 2 | **Analogous** | 3 соседних на круге | 3 |
| 3 | **Complementary** | 2 напротив на круге | 2 (+нейтральные) |
| 4 | **Split-complementary** | База + 2 соседа комплемента | 3 |
| 5 | **Triadic** | 3 равноудалённых (120°) | 3 |
| 6 | **Tetradic** | 2 пары комплементов (прямоугольник) | 4 |
| 7 | **Square** | 4 равноудалённых (90°) | 4 |

## Шаг 4. Собрать палитру по правилу 60-30-10

У тебя есть базовый цвет и схема гармонии. Теперь определи роли цветов.

### Правило 60-30-10

| Роль | Доля | Что это | Пример |
|------|------|---------|--------|
| **Primary (60%)** | Доминирующий | Фон, крупные поверхности | Тёмно-синий фон или белый фон |
| **Secondary (30%)** | Поддерживающий | Карточки, секции, навигация | Светло-серый или мягкий оттенок primary |
| **Accent (10%)** | Акцентный | CTA, ссылки, активные элементы, бейджи | Контрастный яркий цвет |

### Структура UI-палитры

Помимо брендовых цветов, UI нуждается в **семантических** и **нейтральных**:

| Категория | Цвета | Назначение |
|-----------|-------|-----------|
| **Brand primary** | Основной цвет бренда | Логотип, CTA, акценты |
| **Brand secondary** | Дополнительный | Секции, поддержка primary |
| **Neutrals** | Серый (3–5 оттенков), белый, чёрный | Фон, текст, разделители, disabled |
| **Success** | Зелёный | Подтверждение, завершение |
| **Warning** | Жёлтый / янтарный | Предупреждение |
| **Error** | Красный | Ошибка, деструктивное действие |
| **Info** | Синий | Нейтральное уведомление |

**Правило:** семантические цвета должны быть визуально отличимы от брендовых, чтобы не путать «наш синий бренд» с «информационное сообщение».

## Шаг 5. Настроить контраст и accessibility

### WCAG-требования

| Элемент | Минимальный контраст (AA) | Усиленный (AAA) |
|---------|--------------------------|-----------------|
| Обычный текст (<18px) | 4.5 : 1 | 7 : 1 |
| Крупный текст (≥18px bold или ≥24px) | 3 : 1 | 4.5 : 1 |
| UI-компоненты (иконки, границы) | 3 : 1 | — |

### Практика проверки

1. Проверь каждую пару «текст + фон» через contrast checker.
2. Проверь в grayscale — иерархия должна читаться без цвета.
3. Проверь для дальтоников (protanopia, deuteranopia, tritanopia) — не полагайся только на цвет для передачи смысла.
4. Проверь на разных экранах — цвет на OLED, IPS и в печати отличается.

### Инструменты проверки

- **WebAIM Contrast Checker** — быстрая проверка пары.
- **Stark** — плагин для дизайн-инструментов; симуляция дальтонизма.
- **Realtime Colors** — предпросмотр палитры на реальном сайте.
- **Coolors** — генератор с проверкой контраста.
- **Adobe Color** — цветовой круг + accessibility checker.

## Шаг 6. Учесть RGB / CMYK

| Параметр | RGB | CMYK |
|----------|-----|------|
| Среда | Экраны | Печать |
| Модель | Аддитивная (свет) | Субтрактивная (краска) |
| Яркость | Ярче, шире gamut | Тусклее, gamut уже |
| Количество цветов | 16 млн+ | 16 000+ |

**Практические правила:**
- Для экрана: работай в RGB (sRGB или Display P3).
- Для печати: работай сразу в CMYK; не конвертируй из RGB в последний момент.
- Неоновые и очень насыщенные цвета (электрический синий, ярко-фиолетовый) невоспроизводимы в CMYK.
- Логотип создавай в обоих пространствах.

Подробнее: `references/rgb-cmyk-guide.md`.

## Шаг 7. Валидировать палитру

Перед финальной рекомендацией пройди чеклист:

- [ ] **60-30-10:** dominant, secondary и accent визуально сбалансированы?
- [ ] **Контраст:** все пары текст/фон проходят WCAG AA (4.5:1)?
- [ ] **Grayscale-тест:** иерархия читается без цвета?
- [ ] **Дальтонизм:** информация не передаётся только цветом (есть иконки, текст, паттерны)?
- [ ] **Семантика:** success/error/warning не путаются с брендовыми цветами?
- [ ] **Консистентность:** один цвет = одна функция по всему продукту?
- [ ] **Тёмная тема (если нужна):** палитра адаптирована, не просто инвертирована?
- [ ] **Культурный контекст:** символика цвета уместна для целевой аудитории?
- [ ] **Реальный контент:** палитра работает с фотографиями, текстом, иконками — не только в абстрактном swatch?
