# Структура сайта: типы, компоненты и grid basics

## 5 типов структур

### 1. Hierarchical (tree)

Самая распространённая. Главная → категории → подкатегории, как дерево.

**Плюсы:** понятная навигация, хорош для SEO, интуитивен.
**Минусы:** глубокая вложенность = много кликов до контента.
**Best for:** e-commerce, бизнес-сайты, информационные порталы.

### 2. Linear (sequential)

Страницы идут в фиксированном порядке: «Назад» / «Далее».

**Плюсы:** контролируемый путь, отлично для обучения.
**Минусы:** негибкий, не для exploration.
**Best for:** онбординг, туториалы, multi-step forms, курсы.

### 3. Matrix (web)

Страницы связаны сетью ссылок, пользователь выбирает свой путь.

**Плюсы:** свобода исследования, гибкость.
**Минусы:** легко потеряться, сложно поддерживать.
**Best for:** Wiki, базы знаний, портфолио, интерактивные платформы.

### 4. Database (dynamic)

Страницы генерируются из базы данных по запросам и фильтрам.

**Плюсы:** масштабируется на миллионы страниц, персонализация.
**Минусы:** требует бэкенда, сложнее для SEO.
**Best for:** e-commerce с большим каталогом, новостные сайты, CMS.

### 5. Hybrid

Комбинация двух и более моделей.

**Плюсы:** гибкость, адаптация под разные секции сайта.
**Минусы:** сложнее в проектировании и поддержке.
**Best for:** университеты, крупные корпорации, мультибренды.

## Сравнительная таблица

| Структура | Навигация | SEO | Масштабируемость | Сложность |
|-----------|----------|-----|-----------------|-----------|
| Hierarchical | Интуитивная | Отличный | Средняя | Низкая |
| Linear | Контролируемая | Средний | Низкая | Низкая |
| Matrix | Свободная | Сложнее | Средняя | Средняя |
| Database | Фильтры + поиск | Требует усилий | Высокая | Высокая |
| Hybrid | Зависит от mix | Зависит | Высокая | Высокая |

## Компоненты структуры

### Sitemap

Визуальная карта всех страниц и их связей. Рисуется **до** дизайна.

Пример (hierarchical):
```
Homepage
├── Products
│   ├── Category A
│   │   ├── Product 1
│   │   └── Product 2
│   └── Category B
├── About
├── Blog
│   ├── Post 1
│   └── Post 2
├── Contact
└── FAQ
```

### URL-структура

- Чистая, читаемая: `/products/running-shoes/nike-air`
- Отражает иерархию sitemap.
- Без ID и параметров в основных URL.
- Lowercase, дефисы вместо подчёркиваний.

### Internal linking

- Каждая важная страница доступна не более чем за 3 клика от главной.
- Связанные страницы ссылаются друг на друга.
- Помогает SEO (PageRank flow) и навигации.

### Breadcrumbs

- Показывают положение пользователя: `Home > Products > Running Shoes > Nike Air`
- Обязательны для глубоких иерархий (e-commerce).
- Structured data (`BreadcrumbList`) для SEO.

## Grid basics

### Терминология

| Термин | Значение |
|--------|---------|
| **Base unit** | Наименьшая единица (обычно 8px). Всё кратно ей. |
| **Column** | Вертикальный блок контента |
| **Gutter** | Промежуток между колонками |
| **Margin** | Отступ от края viewport |
| **Flowline** | Горизонтальная линия, разделяющая секции |
| **Module** | Пересечение column и row — ячейка контента |

### Hard grid vs Soft grid

| Hard grid | Soft grid |
|-----------|-----------|
| Все элементы строго по baseline | Элементы выровнены по spacing unit |
| Из print-дизайна | Более практичен для web |
| Сложно при responsive | Гибкий, адаптивный |

**Рекомендация:** soft grid для web — более практичен. Не каждый элемент обязан лечь на baseline.

### Размеры для responsive

- **REM** > px (масштабируется с user settings).
- Margins на mobile: маленькие (16–24px), чтобы не тратить место.
- Margins на desktop: liquid (процентные), чтобы текст не растягивался на весь widescreen.
- Gutters: 16–24px (mobile), 24–32px (desktop).
- Колонки: 4 (mobile), 8 (tablet), 12 (desktop) — стандартная 12-column grid.
